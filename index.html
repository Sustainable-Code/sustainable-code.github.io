Sustainable Code

What is Sustainable Code?

Sustainable Code is a software development philosophy based on the Software Craftsmanship Movement. It promotes writing code in such a way that it provides agility for your business as well as a strong work/life balance for the developers.


What are the Principles of Sustainable Code?

  • Easy to modify
  • Easy to test
  • Easy to deploy
  • Easy to understand

Practices Promoted by Sustainable Code

Test Automation

A key to writing Sustainable Code is shortening the feedback loop when writing code. To facilitate this you MUST automate.


  • Unit Testing
    • Unit tests are a critical piece of the test automation infrastructure. They are a low cost way to verify your code complies with the business requirements. It’s also a great way to experiment with different scenarios and see how your code handles it.
  • Integration Testing
    • Integration tests are the next step up from unit tests.
    • These tests verify that multiple methods in your code interact correctly with each other.
    • These tests are more “expensive” than unit tests.
    • Many developers will sometimes confuse unit tests and integration tests.
  • Feature Tests
    • Feature Tests exercise an entire service or UI.
    • These tests verify entire workflows operate as expected.
    • Often times these tests can be slow and in many cases could be better performed by a unit test or integration test.
  • Manual Testing
    • Wait, I thought this section was called “Test Automation?”
      • Testing is not a binary all manual or all automation
      • Exploratory Testing plats a vital role in ensuring code quality.

Automated Deployments

Another key to building Sustainable Code is the ability to deploy at a moment’s notice. To facilitate this you need a reliable, repeatable, automated deployment process.
